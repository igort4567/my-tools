# Верстак

Каркас сайта для онлайн-инструментов (калькуляторы, конвертеры и т.д.)
на Next.js 16 (App Router) + TypeScript + Tailwind CSS v4, готовый к
деплою на Vercel.

## Стек

- **Next.js 16** (App Router, Turbopack) — файловая маршрутизация отлично
  подходит для каталога инструментов: каждый инструмент = отдельная папка.
- **TypeScript**
- **Tailwind CSS v4** — конфигурация в CSS (`app/globals.css`), отдельный
  `tailwind.config.js` не нужен.
- Ноль внешних UI-библиотек — компонентов немного, и все они простые.

## Структура проекта

```
verstak/
├── app/
│   ├── layout.tsx              # общий каркас: шрифты, Header/Footer, метаданные
│   ├── page.tsx                 # главная страница
│   ├── globals.css              # дизайн-токены и глобальные стили (Tailwind v4)
│   ├── icon.svg                 # favicon (подхватывается Next.js автоматически)
│   └── tools/
│       ├── page.tsx             # страница полного каталога /tools
│       └── percent-calculator/
│           └── page.tsx         # пример готового инструмента /tools/percent-calculator
│
├── components/
│   ├── layout/                  # Header, Footer
│   ├── ui/                      # переиспользуемые примитивы: Button, Container
│   └── tools/                   # ToolCard и сами инструменты (PercentCalculator и т.д.)
│
├── lib/
│   ├── tools-config.ts          # реестр всех инструментов — единая точка правды
│   └── utils.ts
│
├── types/
│   └── tool.ts                  # тип Tool
│
└── public/                      # статика: иконки, изображения
```

## Как добавить новый инструмент

Архитектура рассчитана именно на это, поэтому шагов немного:

1. Создайте папку `app/tools/<slug>/page.tsx`. Проще всего скопировать
   `app/tools/percent-calculator/` как образец и переписать содержимое.
2. Если инструменту нужна интерактивность (состояние, обработчики) —
   вынесите логику в клиентский компонент в `components/tools/`
   (с директивой `"use client"` наверху файла), как сделано с
   `PercentCalculator.tsx`, а страницу в `app/tools/<slug>/page.tsx`
   оставьте серверным компонентом — так быстрее грузится и лучше для SEO.
3. Добавьте описание инструмента в массив `tools` в
   `lib/tools-config.ts` — карточка на главной и в `/tools` появится
   автоматически, ничего в компонентах менять не нужно.

Никакого центрального роутера или конфига страниц вести не нужно —
за это отвечает файловая структура `app/tools/`.

## Локальный запуск

Требуется Node.js 20+.

```bash
npm install
npm run dev
```

Сайт будет доступен на http://localhost:3000

## Деплой на Vercel

Проект не требует дополнительной настройки — Vercel сам определяет
Next.js-проект.

1. Залейте проект в Git-репозиторий (GitHub/GitLab/Bitbucket).
2. На vercel.com выберите **Add New → Project** и укажите репозиторий.
3. Framework Preset подставится автоматически ("Next.js"), команды сборки
   менять не нужно.
4. Нажмите **Deploy**.

Либо через CLI:

```bash
npm i -g vercel
vercel
```

## Дизайн

Палитра — бело-зелёная, в духе расчётной миллиметровки: белая бумага,
приглушённая зелёная сетка, один яркий зелёный акцент для действий и
табло с результатом, стилизованное под экран калькулятора. Все цвета и
шрифты заданы токенами в `app/globals.css` (`:root` + `@theme inline`),
их достаточно поменять в одном месте, чтобы перекрасить весь сайт.
